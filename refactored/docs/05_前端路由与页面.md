### 2.3 路由配置设计

- 公开路由设置 `meta: { public: true }`，路由守卫直接放行
- 主布局路由设置 `meta: { requiresAuth: true }`，默认重定向到 `/dashboard`
- 子路由通过 `meta: { title, icon, menu, roles }` 声明页面标题、图标、是否显示菜单、所需角色
- 使用 `createWebHistory()` 模式，路由懒加载 `() => import(...)`
- 404 路由使用 `/:pathMatch(.*)*` 兜底

## 4. 权限路由守卫

### 4.1 路由守卫流程

```
┌─────────────────────────────────────────────────────────────────┐
│                       路由守卫流程                               │
└─────────────────────────────────────────────────────────────────┘

    开始
      │
      ▼
┌─────────────┐     是     ┌─────────────┐
│ 公开路由?   │───────────►│ 直接放行    │
│ (meta.public)│            └─────────────┘
└──────┬──────┘
       │ 否
       ▼
┌─────────────┐     否     ┌─────────────┐
│ 已登录?     │───────────►│ 跳转登录页  │
└──────┬──────┘            └─────────────┘
       │ 是
       ▼
┌─────────────┐     是     ┌─────────────┐
│ 需要角色?   │───────────►│ 检查角色    │
│ (meta.roles)│            └──────┬──────┘
└──────┬──────┘                   │
       │ 否                  ┌────┴────┐
       │                  有权限    无权限
       │                  放行      跳转首页
       └──────────────────────────────────►
```

### 4.2 路由守卫设计

**前置守卫 (beforeEach):**
1. 公开路由 (`meta.public`) 直接放行
2. 检查 Token 是否存在，不存在则跳转登录页
3. 若无用户信息，调用 `authStore.fetchUserInfo()` 获取
4. 检查路由所需角色 (`meta.roles`)，无权限则跳转首页并提示

**后置钩子 (afterEach):**
- 设置页面标题：`${to.meta.title} - 图书管理系统`

**错误处理 (onError):**
- 捕获路由错误，提示"页面加载失败"

### 4.3 权限指令

- 自定义 `v-permission` 指令，在组件挂载时检查当前用户角色
- 使用方式：`<el-button v-permission="['admin', 'librarian']">删除</el-button>`
- 无权限时从 DOM 中移除元素

## 5. 页面布局设计

### 5.1 布局架构

```
┌─────────────────────────────────────────────────────────────────┐
│                      布局架构                                    │
└─────────────────────────────────────────────────────────────────┘

MainLayout (主布局)
│
├── Header (顶部导航)
│   ├── Logo
│   ├── 系统名称
│   ├── 消息通知
│   └── 用户信息下拉菜单
│
├── Sidebar (侧边菜单)
│   ├── 菜单项 (根据权限动态生成)
│   └── 折叠按钮
│
└── Main Content (主内容区)
    ├── Breadcrumb (面包屑)
    ├── Page Title (页面标题)
    ├── Content (页面内容)
    └── Footer (页脚)
```

### 5.2 主布局设计

- 使用 `el-container` + `el-aside` + `el-header` + `el-main` 组合布局
- 侧边栏支持折叠/展开，折叠时宽度 64px，展开时 200px
- 顶部导航左侧放置折叠按钮和面包屑，右侧放置通知和用户下拉菜单
- 主内容区使用 `<router-view>` 渲染子路由，配合 `<transition>` 实现页面切换动画

### 5.3 侧边菜单设计

- 使用 `el-menu` 组件，设置 `router` 属性启用路由模式
- 菜单数据从 `permissionStore.menuRoutes` 获取（已按权限过滤）
- 支持单级菜单 (`el-menu-item`) 和多级菜单 (`el-sub-menu`)
- 菜单项显示图标和标题，通过 `route.meta.icon` 和 `route.meta.title` 配置

### 6.3 组合式函数设计

**useTable<T, Q>** - 通用列表数据管理：
- 接收 `fetchFn` (数据获取函数)、`initialQuery` (初始查询条件)、`immediate` (是否立即加载)
- 返回：`list`, `loading`, `error`, `pagination`, `query`, `fetchData`, `handlePageChange`, `handleSizeChange`, `handleSearch`, `handleReset`
- 自动管理分页状态、加载状态和错误状态

**useForm<T>** - 通用表单管理：
- 管理表单数据、校验状态、提交状态
- 提供 `submit`, `reset`, `validate` 方法

## 7. 状态管理

### 7.1 Pinia Store设计

**auth store (认证状态):**
- State: `token` (localStorage持久化), `user`
- Getters: `isAuthenticated`, `userRole`, `username`
- Actions: `login(credentials)`, `fetchUserInfo()`, `logout()`

**permission store (权限状态):**
- 根据用户角色动态生成可访问的菜单路由
- 提供 `menuRoutes` 计算属性供侧边栏使用
