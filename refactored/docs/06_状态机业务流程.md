# 06 状态机业务流程

## 1. 概述

本文档定义图书管理系统的核心业务状态机及业务流程。所有业务操作都遵循状态机驱动原则，确保数据一致性和业务规则的正确执行。

---

## 2. 状态机设计

### 2.1 借阅记录状态机 (BorrowRecord)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           借阅记录状态机                                     │
└─────────────────────────────────────────────────────────────────────────────┘

                              ┌─────────────┐
                              │   initial   │
                              │   (初始)     │
                              └──────┬──────┘
                                     │ 创建借阅
                                     ▼
┌─────────────┐              ┌─────────────┐              ┌─────────────┐
│  returned   │◄─────────────│  borrowed   │─────────────►│   overdue   │
│  (已归还)    │   正常归还    │  (已借出)    │   逾期检测    │  (已逾期)    │
└─────────────┘              └──────┬──────┘              └──────┬──────┘
      ▲                             │                            │
      │                             │ 标记丢失                    │ 逾期归还
      │                             ▼                            ▼
      │                      ┌─────────────┐              ┌─────────────┐
      │                      │    lost     │              │  returned   │
      │                      │  (已丢失)    │              │  (已归还)    │
      │                      └─────────────┘              └──────┬──────┘
      │                                                          │
      └──────────────────────────────────────────────────────────┘
                              缴纳罚款后归还
```

#### 状态定义

| 状态值 | 中文名称 | 说明 |
|--------|----------|------|
| `borrowed` | 已借出 | 图书已借出，未到应还日期 |
| `overdue` | 已逾期 | 超过应还日期，产生逾期记录 |
| `returned` | 已归还 | 图书已正常归还 |
| `lost` | 已丢失 | 图书被标记为丢失 |

#### 状态转换规则

| 当前状态 | 目标状态 | 触发条件 | 业务规则 |
|----------|----------|----------|----------|
| borrowed | overdue | 定时任务检测 | 当前时间 > due_date |
| borrowed | returned | 用户/管理员归还 | 库存+1，记录归还时间 |
| borrowed | lost | 管理员标记丢失 | 生成罚款记录 |
| overdue | returned | 用户/管理员归还 | 需先处理逾期罚款 |
| overdue | lost | 管理员标记丢失 | 累计罚款+丢失赔偿 |

#### 状态机设计要点

- 使用 `BorrowStateMachine` 类封装状态转换逻辑
- 状态转换表：`{当前状态: {事件: 目标状态}}`
- `can_transition(event)` 检查是否可转换，`transition(event, **context)` 执行转换
- 转换前校验：逾期状态下归还需先处理罚款
- 转换后动作：设置归还时间、触发逾期/丢失事件回调

### 2.2 用户状态机 (User)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                             用户状态机                                       │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────┐
    │ initial │
    │ (初始)   │
    └────┬────┘
         │ 注册
         ▼
┌─────────┐     冻结      ┌─────────┐     多次逾期    ┌─────────────┐
│ active  │──────────────►│ frozen  │───────────────►│ blacklisted │
│ (正常)   │◄──────────────│ (冻结)   │◄───────────────│  (黑名单)    │
└────┬────┘    解冻        └────┬────┘    人工解除     └──────┬──────┘
     │                          │                            │
     │ 注销                     │ 解冻                       │ 人工解除
     ▼                          ▼                            ▼
┌─────────┐               ┌─────────┐                  ┌─────────┐
│inactive │               │  active │                  │  active │
│(已注销)  │               │ (正常)   │                  │ (正常)   │
└─────────┘               └─────────┘                  └─────────┘
```

#### 状态定义

| 状态值 | 中文名称 | 说明 |
|--------|----------|------|
| `active` | 正常 | 用户可正常借阅图书 |
| `frozen` | 冻结 | 用户暂时无法借阅，可归还 |
| `blacklisted` | 黑名单 | 用户被永久禁止借阅 |
| `inactive` | 已注销 | 用户已注销账号 |

#### 状态转换规则

| 当前状态 | 目标状态 | 触发条件 | 业务规则 |
|----------|----------|----------|----------|
| active | frozen | 管理员冻结 | 有未处理违规或异常 |
| active | blacklisted | 系统自动/管理员 | 逾期次数超过阈值 |
| active | inactive | 用户注销 | 无未归还图书和未缴罚款 |
| frozen | active | 管理员解冻 | 问题已解决 |
| frozen | blacklisted | 系统自动 | 冻结期内再次违规 |
| blacklisted | active | 管理员解除 | 特殊情况人工处理 |

#### 状态机设计要点

- 使用 `UserStateMachine` 类封装状态转换逻辑
- 注销前校验：有未归还图书时不允许注销
- 加入黑名单时记录时间和原因
- 每次状态变更记录操作日志

### 2.3 图书状态机 (Book)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                             图书状态机                                       │
└─────────────────────────────────────────────────────────────────────────────┘

    ┌─────────┐
    │ initial │
    │ (初始)   │
    └────┬────┘
         │ 录入
         ▼
┌─────────────┐    标记不可借    ┌─────────────┐
│  available  │◄────────────────►│ unavailable │
│   (可借)     │    恢复可借      │  (不可借)    │
└──────┬──────┘                  └─────────────┘
       │
       │ 标记丢失
       ▼
┌─────────────┐
│    lost     │
│   (丢失)     │
└─────────────┘
```

#### 状态定义

| 状态值 | 中文名称 | 说明 |
|--------|----------|------|
| `available` | 可借 | 图书可正常借阅 |
| `unavailable` | 不可借 | 图书暂不可借阅（维修、预留等） |
| `lost` | 丢失 | 图书已丢失 |

#### 状态转换规则

| 当前状态 | 目标状态 | 触发条件 | 业务规则 |
|----------|----------|----------|----------|
| available | unavailable | 管理员标记 | 库存必须全部归还 |
| available | lost | 管理员标记 | 生成丢失记录 |
| unavailable | available | 管理员恢复 | - |

#### 状态机设计要点

- 使用 `BookStateMachine` 类封装状态转换逻辑
- 标记不可借时校验：所有副本必须已归还（total_copies == available_copies）
- 标记丢失时需处理未归还的借阅记录

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                             罚款状态机                                       │

    ┌─────────┐
    │ initial │
    │ (初始)   │
    └────┬────┘
         │ 生成罚款
         ▼
┌─────────────┐     缴纳      ┌─────────────┐

┌─────────────┐
│   waived    │
│  (已免除)    │
└─────────────┘
```

#### 状态定义

| 状态值 | 中文名称 | 说明 |
|--------|----------|------|
| `pending` | 待缴纳 | 罚款待用户缴纳 |
| `paid` | 已缴纳 | 罚款已缴纳完成 |
| `waived` | 已免除 | 罚款被管理员免除 |

#### 状态转换规则

| 当前状态 | 目标状态 | 触发条件 | 业务规则 |
|----------|----------|----------|----------|
| pending | paid | 用户缴纳 | 记录缴纳时间和金额 |
| pending | waived | 管理员免除 | 记录免除原因和操作人 |

#### 状态机设计要点

- 使用 `OverdueStateMachine` 类封装状态转换逻辑
- 缴纳时记录 `paid_at` 和 `paid_amount`
- 免除时必须提供原因，记录 `waived_at`, `waived_by`, `waive_reason`

#### 借书服务设计

**BorrowService.borrow_book(user_id, book_id, operator_id)**

校验步骤：
1. **用户校验**: 用户存在、非黑名单、非冻结、非注销
2. **图书校验**: 图书存在、状态可借、库存 > 0
3. **重复借阅校验**: 同一用户未借阅同一本书（borrowed/overdue 状态）
4. **借阅配额校验**: 当前借阅数 < max_books（优先使用用户自定义配额，否则使用系统规则）
5. **执行借书**: 使用行级锁扣减库存 -> 根据规则计算应还日期 -> 创建借阅记录 -> 提交事务

#### 还书服务设计

**ReturnService.return_book(borrow_id, operator_id)**

处理步骤：
1. **查询借阅记录**: 加载关联的逾期记录
2. **状态校验**: 非 returned/lost 状态才可归还
3. **逾期检测**: 当前时间 > due_date 且状态非 overdue 时，生成逾期记录
4. **罚款检查**: 有未缴纳罚款时拒绝归还，提示先缴纳
5. **执行归还**: 使用状态机转换状态 -> 增加库存 -> 设置归还时间 -> 提交事务

#### 逾期处理服务设计

**OverdueService.check_overdue_batch()** - 定时任务批量检测：
1. 查询所有 status=borrowed 且 due_date < now 的记录
2. 逐条处理：更新状态为 overdue、计算逾期天数和罚款（扣除宽限期 grace_days）、创建逾期记录、更新用户逾期次数
3. 检查黑名单阈值：逾期次数 >= threshold 的用户自动加入黑名单

**OverdueService.pay_fine(overdue_id, user_id)** - 缴纳罚款：
1. 校验逾期记录存在、属于当前用户、状态为 pending
2. 使用状态机转换状态为 paid
3. 记录缴纳时间和金额

### 3.4 黑名单自动规则

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                          黑名单自动规则                                      │
└─────────────────────────────────────────────────────────────────────────────┘

1. 逾期次数阈值触发
   ├── 条件: 用户累计逾期次数 >= blacklist_threshold (默认3次)
   ├── 检测时机: 
   │   ├── 每日定时任务扫描
   │   └── 每次新增逾期记录时
   └── 动作: 自动加入黑名单

2. 严重逾期触发
   ├── 条件: 单条借阅逾期天数 >= 90天
   ├── 检测时机:
   │   └── 每日定时任务扫描
   └── 动作: 自动加入黑名单 + 生成高额罚款

3. 图书丢失触发
   ├── 条件: 用户累计丢失图书 >= 2本
   ├── 检测时机:
   │   └── 标记图书丢失时
   └── 动作: 自动加入黑名单
```

#### 黑名单服务设计

**BlacklistService.add_to_blacklist(user_id, operator_id, reason, auto_triggered)**:
- 校验用户存在且未在黑名单中
- 使用 UserStateMachine 执行状态转换
- 自动触发时在原因前加 `[系统自动]` 前缀
- 记录操作日志

**BlacklistService.remove_from_blacklist(user_id, operator_id, reason)**:
- 校验用户在黑名单中
- 检查解除条件：所有图书已归还、所有罚款已缴纳
- 使用 UserStateMachine 执行状态转换
- 重置用户逾期计数
- 记录操作日志

**BlacklistService.check_auto_blacklist_conditions(user_id)**:
- 检查逾期次数是否达到阈值
- 检查是否存在严重逾期（>90天）
- 返回触发原因或 None

## 4. 状态变更日志

所有状态变更都应记录详细的操作日志，用于审计和追溯。

**AuditService.log_state_transition()** 记录内容：
- 操作用户信息 (user_id, username)
- 目标对象信息 (target_type, target_id, target_name)
- 操作类型 (action)
- 状态变更详情 (old_state -> new_state)
- 变更原因 (reason)
- 额外元数据 (metadata)
